Getting Started

Download, install, and compress your first video in under two minutes.

1. Download

Grab the latest .dmg from the Releases page. VideoOptim runs on macOS 12 or later, Apple Silicon only (M1 and newer).

Open the downloaded .dmg, drag VideoOptim into your Applications folder, then eject the disk image.

2. First launch

VideoOptim is not notarized, so macOS Gatekeeper may block it on first open. There are two ways to bypass this:

Option A — Right-click to open

In Finder, right-click (or Control-click) VideoOptim.app and choose Open. macOS will ask for confirmation once; after that it opens normally.

Option B — Remove the quarantine flag

Open Terminal and run:

xattr -cr /Applications/VideoOptim.app

Then launch the app normally from Launchpad or Finder.

ℹ

You only need to do this once. After the first successful launch, macOS remembers your choice and opens VideoOptim without prompting.

3. Add videos

VideoOptim accepts files in three ways:

A

Drag and drop

Drag video files or an entire folder onto the VideoOptim window. A blue overlay appears while dragging — release to add the files.

B

File picker

Click Choose files… in the empty state, or Add files… from the toolbar when the list already has jobs.

C

Folder picker

Click Choose folder… to add all supported video files inside a folder (and subfolders). Useful for bulk batch jobs.

ℹ

Supported formats: MP4, MOV, MKV, AVI, WebM. Files already encoded in H.265 (HEVC) are skipped automatically — re-encoding them would make them larger, not smaller.

4. Compression runs automatically

Once files are added, VideoOptim starts encoding immediately — one file at a time. Each row shows a live progress bar, elapsed time, and the current encode speed.

You can control the queue at any time using the control bar that appears while encoding:

⏸

Pause

Suspends the current ffmpeg process. The file stays in place, no data is lost. Resume picks up exactly where it left off.

▶

Resume

Continues the paused encode.

⏹

Stop

Cancels the current file and clears all waiting jobs. Any already-completed files remain in the list.

5. Output files

Compressed files are saved in the same folder as the original, with an _optimized.mp4 suffix added to the filename:

holiday.mov → holiday_optimized.mp4 interview.mkv → interview_optimized.mp4

The original file is never modified or deleted automatically. VideoOptim keeps both until you decide what to do.

ℹ

No gain discard — if the compressed output ends up larger than the original (common with already-compressed or very short clips), VideoOptim discards the output and keeps the original. You'll see "No gain" in the savings column. This behaviour can be turned off in Settings.

6. Review and clean up

When encoding finishes, the summary bar shows total space saved across all files. Review the results — right-click any row to reveal a file in Finder or open it.

When you're happy with the results, click Clean up originals in the toolbar. This moves all original files that have a smaller compressed version to the macOS Trash. Nothing is permanently deleted — you can restore them from Trash if needed.

7. Settings

Open Settings with ⌘, or the gear icon in the toolbar.

Setting Default Notes
Encoder VideoToolbox Hardware-accelerated HEVC. Fastest option, uses Apple Silicon media engine.
libx265 Software encoder. Slower, but produces smaller files at the same visual quality.
CRF 24 Quality level for libx265 only. Lower = better quality, larger file. Range 18–35.
Keep audio On Copies the audio track without re-encoding. Turning this off strips audio entirely.
Discard if no gain On Deletes the output and keeps the original when the compressed file is larger.
Accepted formats All Toggle which file extensions are added when dropping folders.
